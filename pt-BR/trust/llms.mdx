Não há nenhum prompt de copiar/colar para manter. Para ensinar a uma IA tudo sobre o Pilot Status, aponte-a para as fontes canônicas legíveis por máquina.

## llms.txt

A referência principal para ferramentas de IA fica no domínio da documentação:

```text
https://pilotstatus.com.br/llms.txt
```

Ele indexa endpoints, schemas de webhook, conceitos e códigos de erro. Cole a URL no seu agente de código ou ferramenta de LLM e ele conseguirá navegar até as páginas de documentação detalhada que precisar — cada tópico está vinculado a partir do índice, então o modelo lê apenas o que é relevante, em vez de um prompt gigante.

<Tip>
  Ao pedir a um assistente de código para construir uma integração com o Pilot Status, forneça primeiro a URL do `llms.txt`. Ele vai extrair os formatos exatos de request/response (por exemplo, os três modos de `POST /v1/messages/send`) em vez de alucinar endpoints.
</Tip>

## Servidor MCP

Para agentes que devem **agir** (enviar mensagens, ler conversas, gerenciar números) em vez de apenas ler a documentação, use o servidor oficial do Model Context Protocol — um conector hospedado em `https://mcp.pilotstatus.com.br/mcp` que expõe a API pública `/v1` como ferramentas, com autenticação via OAuth ou `x-api-key`.

Consulte [Servidor MCP](/pt-BR/integrations/mcp-server) para a lista de ferramentas e configuração, e [Agentes de IA](/pt-BR/guides/ai-agents) para padrões de bots orientados por webhook.
