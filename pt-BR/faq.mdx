<AccordionGroup>
  <Accordion title="O que é o Pilot Status?">
    O Pilot Status é uma plataforma de infraestrutura de WhatsApp criada para desenvolvedores e equipes de produto. Ela gerencia tudo o que é difícil em rodar o WhatsApp em escala — conexões persistentes, rotação de proxy, gerenciamento de sessões e entrega — para que você possa focar no seu produto em vez da sua infraestrutura.

    Você interage com o Pilot Status por meio de uma API REST padrão e recebe eventos em tempo real via webhooks. O painel oferece à sua equipe uma interface visual para gerenciar números, navegar por logs e conversar ao vivo com contatos.
  </Accordion>

  <Accordion title="Preciso de um servidor ou VPS para usar o Pilot Status?">
    Não. O Pilot Status roda e gerencia toda a infraestrutura para você. Não há servidores para provisionar, sessões de WhatsApp para manter ativas nem proxies para configurar. Você envia requisições à API e recebe webhooks — essa é toda a interface. Isso também significa que você tem redundância e gerenciamento de uptime embutidos, sem nenhum esforço de DevOps do seu lado.
  </Accordion>

  <Accordion title="Qual é a diferença entre a API oficial e a não oficial do WhatsApp?">
    A **API oficial** (Meta Cloud API) é suportada diretamente pela Meta. Ela exige uma conta do WhatsApp Business verificada, usa templates de mensagem aprovados pela Meta para conversas de saída e oferece as maiores garantias de entregabilidade e conformidade. É a escolha certa para setores regulados ou para mensagens de marketing e transacionais em larga escala.

    A **API não oficial** conecta por meio de leitura de um QR code, assim como o WhatsApp Web. Ela não exige uma conta Meta Business, oferece suporte a tipos de mensagem mais flexíveis e é mais rápida de configurar. Ela não é oficialmente endossada pela Meta, então é mais adequada para ferramentas internas, protótipos e produtos onde o processo de aprovação da Meta não é um requisito.

    O Pilot Status oferece suporte a ambas — todos os planos incluem conexões via Meta Cloud API e Coexistence junto com a API não oficial (as cobranças por conversa da própria Meta são faturadas pela Meta separadamente).
  </Accordion>

  <Accordion title="Posso migrar de outro provedor sem reescrever código?">
    Sim. O Pilot Status inclui uma **API Layer** que é compatível com os formatos de API de WhatsApp mais comuns. Para migrar, você altera a base URL na sua integração existente para apontar para o Pilot Status — a estrutura da sua requisição, o padrão de autenticação e o handler de webhook permanecem os mesmos. A maioria das equipes conclui uma migração em menos de uma hora.
  </Accordion>

  <Accordion title="Existe um plano gratuito?">
    Sim. O plano Free oferece um número conectado, até 200 mensagens no total, logs de mensagens e webhooks — sem necessidade de cartão de crédito. Ele foi projetado para avaliação e desenvolvimento em estágio inicial. Quando você estiver pronto para produção, pode fazer upgrade para um plano pago diretamente pelo painel. Consulte a página [Planos](/pt-BR/plans) para uma comparação completa.
  </Accordion>

  <Accordion title="Meus dados e as mensagens dos meus usuários estão seguros?">
    Sim. As mensagens enviadas pelo Pilot Status trafegam com a criptografia de ponta a ponta do WhatsApp, a mesma criptografia que protege todas as conversas do WhatsApp. O Pilot Status armazena logs de mensagens para que você tenha um histórico auditável — e você pode controlar ou desabilitar esse armazenamento por número com os [modos de retenção de PII](/pt-BR/trust/data-retention).

    Suas chaves de API têm escopo para seus números e devem ser mantidas privadas. Você pode regenerá-las a qualquer momento na página **API Keys** do painel.
  </Accordion>

  <Accordion title="Recebo eventos de mensagens em tempo real?">
    Sim. O Pilot Status dispara eventos de webhook em milissegundos após cada evento de mensagem — mensagens de entrada, confirmações de entrega (sent, delivered, read) e mudanças de status de conexão. Você cria webhooks na página **Webhooks** do painel (ou via `POST /v1/webhooks`), e o Pilot Status envia uma requisição POST para a sua URL cada vez que um evento assinado ocorre. Para detalhes completos do payload, consulte a documentação de [Webhooks](/pt-BR/concepts/webhooks).
  </Accordion>

  <Accordion title="O Pilot Status funciona com agentes de IA e ferramentas de automação como o n8n?">
    Sim. Como o Pilot Status expõe uma API REST padrão e dispara webhooks via HTTP, ele funciona com qualquer ferramenta que consiga fazer requisições HTTP ou escutar webhooks. Isso inclui plataformas de automação como **n8n** (existe um node oficial da comunidade) e **Make**, frameworks de IA como **LangChain** e **GPT Maker**, e backends personalizados construídos em qualquer linguagem.

    O Pilot Status também oferece um **servidor MCP** hospedado em `https://mcp.pilotstatus.com.br/mcp` para frameworks de agentes de IA que suportam o Model Context Protocol. Consulte [Agentes de IA](/pt-BR/guides/ai-agents).
  </Accordion>
