Se você já executa automação de WhatsApp na Evolution API — seja Evolution GO ou Evolution V2 — pode migrar para o Pilot Status sem tocar em uma única linha do código da sua aplicação. A **API Layer** do Pilot Status é uma camada de compatibilidade que aceita requisições no formato exato que sua integração existente já envia, as traduz internamente e as roteia pela infraestrutura do Pilot Status. Seus corpos de requisição, formatos de resposta, payloads de webhook e cabeçalhos de autenticação permanecem os mesmos. A única coisa que muda é o hostname na sua base URL.

## O que é a API Layer

A API Layer fica na frente da plataforma Pilot Status e fala o mesmo dialeto HTTP dos provedores de WhatsApp que sua equipe já conhece. Quando sua aplicação envia uma requisição ao endpoint da API Layer, o Pilot Status a interpreta usando o schema específico do provedor, executa a operação em sua infraestrutura gerenciada e retorna uma resposta no formato que seu código espera.

Isso significa que você obtém todos os benefícios do Pilot Status — proxies gerenciados, sem servidores auto-hospedados, integrações oficiais com a Meta, escalonamento automático e SLAs de uptime — sem um sprint de migração, uma reescrita ou um ciclo de QA para validar novos formatos de requisição.

## O que permanece igual

Quando você migra via API Layer, o seguinte permanece completamente inalterado na sua aplicação:

- **Formato do corpo da requisição** — os payloads JSON que você envia hoje funcionam sem modificação.
- **Formato da resposta** — nomes de campos, tipos e estrutura são idênticos ao que você já processa.
- **Formato do payload de webhook** — nomes de eventos e schemas de payload correspondem aos seus handlers existentes.
- **Cabeçalhos de autenticação** — continue usando o mesmo nome e formato de cabeçalho que seu provedor exige (o Pilot Status o mapeia internamente).
- **Caminhos e métodos** — todo caminho de endpoint e método HTTP que você chama atualmente é suportado no mesmo caminho relativo sob a nova base URL.

## O que muda

| O que | Antes | Depois |
| --- | --- | --- |
| **Base URL** | Seu servidor auto-hospedado | `https://pilotstatus.com.br/api/layer/{provider}/` |
| **Cobrança** | Custos do seu servidor / VPS | Assinatura do Pilot Status |
| **Infraestrutura** | Servidor, proxies e atualizações auto-gerenciados | Totalmente gerenciado pelo Pilot Status |
| **Responsabilidade de uptime** | Sua | SLA do Pilot Status |

<Note>
