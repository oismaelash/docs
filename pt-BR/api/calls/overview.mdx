# Chamadas do WhatsApp Business — visão geral

Receba e faça **chamadas de voz** do WhatsApp pela API pública. Duas direções:

- **UIC** (chamada iniciada pelo usuário) — o cliente liga para o seu número do WhatsApp. Gratuita.
- **BIC** (chamada iniciada pela empresa) — o seu sistema liga para o cliente. Requer uma **permissão de chamada** concedida previamente pelo usuário.

<Warning>
  **Somente números da Meta Cloud API.** Em números não oficiais (pareados por QR), todos os endpoints de chamadas retornam `400` com `code: "FEATURE_NOT_SUPPORTED"`. Todos os endpoints de chamadas exigem uma chave de API **com escopo de número** (`x-api-key: ps_...`).
</Warning>

## Arquitetura apenas de sinalização

O Pilot Status é **apenas de sinalização**: a API retransmite SDPs (RFC 8866); **o áudio flui diretamente entre o seu navegador/cliente WebRTC e o WhatsApp** — ele nunca passa pelos servidores do Pilot Status.

O seu lado da integração:

1. Seu cliente WebRTC produz um SDP **offer** (BIC) ou SDP **answer** (UIC).
2. Você troca esses SDPs por meio da [API de Chamadas](/pt-BR/api/calls/endpoints) (`POST /v1/calls`, `accept`, `pre-accept`, …).
3. Uma vez conectado, o áudio transita ponto a ponto entre o cliente e o WhatsApp.

## Softphone no painel — sem necessidade de código WebRTC

No painel (`/chat`) você pode **atender e fazer chamadas direto do navegador** sem nenhuma implementação WebRTC própria. Use a API apenas se quiser ter chamadas dentro do seu próprio produto.

## Cobrança

<Note>
  - As chamadas **BIC** são cobradas pela **Meta diretamente na WABA** — por minuto, em **pulsos de 6 segundos**, e **apenas quando a chamada é atendida**.
  - As chamadas **UIC** são **gratuitas**.
  - **O Pilot Status não cobra nada pelas chamadas.**
</Note>

## Eventos de webhook

Inscreva seu webhook em `call.ringing`, `call.connected`, `call.ended`, `call.missed`, `call.permission_updated` — ou `"*"`. Os payloads normalizados `call.*` são **planos** (sem invólucro `data`); o **envelope nativo `calls` da Meta** (bruto `entry[].changes[].value` com os SDPs) também está disponível para sinalização WebRTC personalizada. Consulte a [referência de eventos](/pt-BR/api/webhooks/events).
