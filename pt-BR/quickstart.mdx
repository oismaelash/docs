Este guia orienta você por tudo o que precisa para sair do zero e enviar sua primeira mensagem no WhatsApp pela API do Pilot Status. Você vai criar uma conta, conectar um número, pegar sua chave de API e fazer uma chamada de API ao vivo — sem necessidade de configurar servidor ou proxy.

<Steps>
  <Step title="Crie sua conta">
    Acesse [pilotstatus.com.br](https://pilotstatus.com.br) e cadastre-se para uma conta gratuita. O plano gratuito inclui um número de WhatsApp e até 200 mensagens — sem necessidade de cartão de crédito.
  </Step>
  <Step title="Conecte seu número">
    No painel, clique em **Conectar número** e escolha seu método de conexão:

    - **QR Code (não oficial):** Abra o WhatsApp no seu celular, toque em **Aparelhos conectados → Conectar um aparelho** e escaneie o QR code exibido no painel.
    - **Credenciais da Meta (oficial):** Selecione o modo Cloud API ou Coexistência, depois insira suas credenciais do Meta Business e o ID da sua Conta do WhatsApp Business.

    Seu número aparecerá como **Conectado** assim que o handshake for concluído. Para um passo a passo completo de ambos os métodos, veja [Conectar um Número de WhatsApp](/pt-BR/connecting-numbers).
  </Step>
  <Step title="Obtenha sua chave de API">
    No painel, abra a página **API Keys** (`/api-keys`) e copie a chave do seu número conectado — ela começa com `ps_`. Mantenha esse valor em segredo — ele autentica cada requisição de API que você faz.
  </Step>
  <Step title="Envie sua primeira mensagem">
    Faça a requisição a seguir, substituindo `ps_your_key_here` pela sua chave e `destinationNumber` por um número de WhatsApp real no qual você possa verificar a entrega. A própria chave identifica qual dos seus números envia a mensagem.

    <CodeGroup>

    ```bash cURL
    curl -X POST https://pilotstatus.com.br/v1/messages/send \
      -H "x-api-key: ps_your_key_here" \
      -H "Content-Type: application/json" \
      -d '{
        "destinationNumber": "+5511999999999",
        "text": "Hello from Pilot Status! 👋"
      }'
    ```

    </CodeGroup>

    Uma resposta bem-sucedida retorna `HTTP 202` com um `id` e um `status` igual a `QUEUED`. O Pilot Status entrega a mensagem e dispara seu webhook conforme os eventos de entrega e leitura chegam. O mesmo endpoint também envia templates (`templateId` \+ `variables`) e mídia (`media` \+ `mediaType`).

    <Tip>
      Os números de telefone devem estar no formato E.164 **com o `+` no início** — por exemplo, `+5511999999999` para um número brasileiro (\+55), DDD 11.
    </Tip>
  </Step>
  <Step title="Receba eventos">
    Para receber confirmações de entrega em tempo real, notificações de leitura e mensagens recebidas, crie um webhook na página **Webhooks** do painel (ou via `POST /v1/webhooks`). O Pilot Status fará um POST com um payload JSON para sua URL em milissegundos após cada evento. Você pode usar um serviço como o [Webhook.site](https://webhook.site) para inspecionar os payloads durante o desenvolvimento.
  </Step>
</Steps>
