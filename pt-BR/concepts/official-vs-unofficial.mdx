O Pilot Status oferece duas formas distintas de conectar um número de WhatsApp: a **API Oficial** (baseada na Cloud API da Meta ou no modo Coexistência) e a **API Não Oficial** (baseada no protocolo do WhatsApp Web). Você escolhe o modo de conexão por número, então pode usar os dois tipos dentro do mesmo projeto, dependendo das suas necessidades.

## API Oficial

A API Oficial se integra diretamente à infraestrutura da Meta, oferecendo um canal compatível e pronto para produção para mensagens no WhatsApp. Há duas variantes disponíveis no Pilot Status:

**Cloud API** é a solução de mensagens hospedada pela Meta. Você conecta sua Conta do WhatsApp Business (WABA), envia seu número remetente para verificação e começa a enviar. Todo o tráfego de mensagens flui pelos servidores da Meta, o que significa que a entrega é altamente confiável e sua integração é totalmente compatível com o WhatsApp.

**Modo Coexistência** é uma variante especial da API Oficial que permite que seu aplicativo WhatsApp Business continue rodando no seu celular ao mesmo tempo que a integração via API. Sua equipe pode continuar usando o aplicativo normalmente enquanto seu backend envia e recebe mensagens de forma programática através do Pilot Status — sem precisar escolher entre um ou outro.

<Tip>
  O modo Coexistência é ideal se você quiser continuar usando o aplicativo WhatsApp Business no seu celular e, ao mesmo tempo, integrar via API.
</Tip>

Como a API Oficial roteia pela Meta, todas as mensagens enviadas para novas conversas incorrem nas taxas por conversa da Meta. Essas cobranças variam por país e categoria de mensagem e são faturadas diretamente pela Meta — elas não fazem parte da sua assinatura do Pilot Status.

## API Não Oficial

A API Não Oficial usa o protocolo do WhatsApp Web para enviar e receber mensagens sem exigir uma conta Meta Business. Você conecta um número simplesmente escaneando um QR code no painel do Pilot Status, da mesma forma que faria para vincular o WhatsApp Web em um navegador.

Uma vez conectado, o Pilot Status gerencia tudo nos bastidores: um proxy dedicado e totalmente gerenciado mantém sua sessão estável e cuida dos detalhes de conexão de baixo nível. Você nunca provisiona nem mantém nenhuma infraestrutura por conta própria.

A API Não Oficial libera recursos que a API Oficial não suporta, incluindo o envio de mensagens para **grupos**, o gerenciamento de **canais de newsletter** e o envio de mensagens interativas ricas sem pré-aprovação de templates. Variáveis e conteúdo dinâmico funcionam exatamente da mesma forma que na API Oficial.

<Note>
  A API não oficial usa o protocolo do WhatsApp Web. Embora amplamente utilizado, ele não é oficialmente suportado pela Meta.
</Note>

## Escolhendo o Modo Certo

Use a orientação abaixo para escolher o tipo de conexão que se encaixa no seu caso de uso.

<Accordion title="Envio de mensagens reguladas (bancos, saúde, finanças)">
  Escolha a **API Oficial**. Setores regulados exigem um canal de mensagens compatível e auditável. A API Oficial é aprovada pelo WhatsApp e oferece a rastreabilidade e a confiabilidade que esses ambientes exigem.
</Accordion>

<Accordion title="Marketing de alto volume com templates aprovados">
  Escolha a **API Oficial**. A categoria de template Marketing da Meta foi projetada para campanhas promocionais em escala. A aprovação garante que suas mensagens sejam entregues de forma confiável e que sua conta permaneça em boa situação.
</Accordion>

<Accordion title="Prototipagem rápida ou ferramentas internas">
  Escolha a **API Não Oficial**. Você pode estar funcionando em segundos — sem conta Meta, sem envio de template, sem espera por aprovação. É o caminho mais rápido do zero a uma integração de WhatsApp funcional.
</Accordion>

<Accordion title="Agentes de IA e chatbots">
  Ambos os modos funcionam bem. Se você precisa entrar no ar rapidamente ou quer evitar o processo de aprovação da Meta, a API Não Oficial é mais fácil de começar. Se o seu chatbot lida com dados sensíveis ou precisa operar em escala empresarial, a API Oficial oferece uma base mais duradoura.
</Accordion>

<Accordion title="Produtos SaaS com números de propriedade do cliente">
  Ambos os modos são suportados. A API Não Oficial tende a ser mais fácil para o onboarding de clientes — seus usuários simplesmente escaneiam um QR code em vez de configurar uma conta Meta Business. Você sempre pode migrar números individuais para a API Oficial mais tarde, conforme as necessidades dos clientes crescem.
</Accordion>
