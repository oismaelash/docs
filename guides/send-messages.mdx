The Pilot Status REST API lets you send WhatsApp messages programmatically — no server setup, proxy management, or WhatsApp Business infrastructure required. You authenticate every request with your API key and specify which connected number to send from using its `number_id`. This guide walks you through sending text messages, approved templates, and interactive button messages.

## Send a text message

To send a plain text message, make a `POST` request to `/v1/messages/text` with the recipient's phone number (including country code, without the `+` sign) and the message body.

<CodeGroup>

```bash curl
curl -X POST https://api.pilotstatus.com.br/v1/messages/text \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "number_id": "num_abc123",
    "to": "5511999999999",
    "text": "Hello from Pilot Status! 👋"
  }'
```

```javascript Node.js (fetch)
const response = await fetch('https://api.pilotstatus.com.br/v1/messages/text', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    number_id: 'num_abc123',
    to: '5511999999999',
    text: 'Hello from Pilot Status! 👋'
  })
});

const data = await response.json();
console.log(data);
```

</CodeGroup>

A successful request returns the message ID and its initial status:

```json
{
  "id": "msg_xyz789",
  "status": "sent",
  "to": "5511999999999",
  "timestamp": "2024-01-15T09:41:00Z"
}
```

Save the `id` if you want to correlate delivery and read events from your webhook later.

## Send a template message (official API)

Template messages are required when you initiate a conversation with a user outside the 24-hour messaging window on the official WhatsApp Cloud API or Coexistence integration. Templates must be pre-approved by Meta before use.

<CodeGroup>

```bash curl
curl -X POST https://api.pilotstatus.com.br/v1/messages/template \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "number_id": "num_abc123",
    "to": "5511999999999",
    "template": {
      "name": "order_confirmation",
      "language": "en_US",
      "components": [
        {
          "type": "body",
          "parameters": [
            { "type": "text", "text": "Maria" },
            { "type": "text", "text": "#ORD-4521" },
            { "type": "text", "text": "January 15, 2024" }
          ]
        }
      ]
    }
  }'
```

```javascript Node.js (fetch)
const response = await fetch('https://api.pilotstatus.com.br/v1/messages/template', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    number_id: 'num_abc123',
    to: '5511999999999',
    template: {
      name: 'order_confirmation',
      language: 'en_US',
      components: [
        {
          type: 'body',
          parameters: [
            { type: 'text', text: 'Maria' },
            { type: 'text', text: '#ORD-4521' },
            { type: 'text', text: 'January 15, 2024' }
          ]
        }
      ]
    }
  })
});

const data = await response.json();
console.log(data);
```

</CodeGroup>

<Note>
  Template messages are only required for the official Cloud API and Coexistence integrations. Unofficial (QR-code) connections can send free-form messages at any time.
</Note>

## Send an interactive message with buttons

Interactive messages let recipients tap a button instead of typing a reply — ideal for confirmations, menus, and chatbot flows. Use quick-reply buttons for up to three response options.

<CodeGroup>

```bash curl
curl -X POST https://api.pilotstatus.com.br/v1/messages/interactive \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "number_id": "num_abc123",
    "to": "5511999999999",
    "body": "Your order #ORD-4521 is ready for pickup. Would you like to confirm collection?",
    "buttons": [
      { "type": "reply", "id": "confirm_yes",  "label": "✅ Yes, confirm" },
      { "type": "reply", "id": "confirm_no",   "label": "❌ Not yet" },
      { "type": "reply", "id": "confirm_help", "label": "💬 Contact support" }
    ]
  }'
```

```javascript Node.js (fetch)
const response = await fetch('https://api.pilotstatus.com.br/v1/messages/interactive', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    number_id: 'num_abc123',
    to: '5511999999999',
    body: 'Your order #ORD-4521 is ready for pickup. Would you like to confirm collection?',
    buttons: [
      { type: 'reply', id: 'confirm_yes',  label: '✅ Yes, confirm' },
      { type: 'reply', id: 'confirm_no',   label: '❌ Not yet' },
      { type: 'reply', id: 'confirm_help', label: '💬 Contact support' }
    ]
  })
});

const data = await response.json();
console.log(data);
```

</CodeGroup>

When the user taps a button, Pilot Status fires a `message.received` webhook to your endpoint with the button `id` in the payload — use it to branch your conversation logic.

<Tip>
  Use webhooks to track when messages are delivered and read. See the [Receive Messages guide](/guides/receive-messages) to set up your webhook endpoint and handle delivery events.
</Tip>

## Rate limits and best practices

Pilot Status enforces per-number rate limits to keep your account in good standing with WhatsApp. Keep the following in mind as you build:

**Don't hardcode `number_id`** — store it in an environment variable or configuration file so you can rotate or swap numbers without a code deployment.

**Handle errors gracefully** — the API returns standard HTTP status codes:

| Status | Meaning | What to do |
|--------|---------|------------|
| `400` | Invalid phone number or malformed request | Validate the `to` field before sending; log and skip |
| `401` | Missing or invalid API key | Check your `Authorization` header |
| `429` | Rate limit exceeded | Back off and retry after the `Retry-After` header value |
| `5xx` | Server error | Retry with exponential backoff |

**Queue outbound messages** — if you're sending to many recipients, use a job queue rather than firing all requests in parallel. This prevents hitting the rate limit and makes retries easier to manage.

**Use templates for cold outreach** — on the official API, always use Meta-approved templates when messaging users who haven't contacted you in the last 24 hours to avoid message rejection.
