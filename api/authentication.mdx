Every request you make to the Pilot Status API must be authenticated with an API key. You pass your key as a Bearer token in the `Authorization` header — there are no cookies, session tokens, or OAuth flows involved. This page walks you through generating your key, attaching it to requests, keeping it secure, and handling authentication errors.

## Getting Your API Key

You generate and manage API keys directly from the Pilot Status dashboard. Follow these steps to obtain your key:

1. Log in to your Pilot Status dashboard at [pilotstatus.com.br](https://pilotstatus.com.br).
2. Navigate to **Settings → API Keys**.
3. Click **Generate Key** to create a new key, or copy your existing key if one is already present.
4. Store the key securely — it will not be shown again after its initial creation.

<Warning>
  Never commit API keys to version control. Use a secrets manager or environment variables in production.
</Warning>

## Using Your API Key

Include your API key in the `Authorization` header of every request using the `Bearer` scheme. The header must be present on all calls — including `GET` requests that don't have a body.

**cURL example:**

```bash
curl https://api.pilotstatus.com.br/v1/numbers \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Node.js example:**

```javascript
const response = await fetch('https://api.pilotstatus.com.br/v1/numbers', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  }
});
```

<Note>
  Replace `YOUR_API_KEY` with the actual key you copied from the dashboard. The word `Bearer` followed by a single space must appear exactly as shown — omitting it or changing the casing will result in a `401` error.
</Note>

## Security Best Practices

Keeping your API key secure protects your WhatsApp numbers and message data. Follow these practices in every environment.

- **Never expose your key in client-side code or public repositories.** API keys embedded in browser JavaScript or committed to a public repo can be harvested and abused.
- **Use environment variables.** Reference your key as `process.env.PILOTSTATUS_API_KEY` in Node.js (or the equivalent in your language/framework) rather than hardcoding it in source files.
- **Rotate keys periodically.** Visit **Dashboard → Settings → API Keys** to generate a fresh key on a regular schedule and update your services accordingly.
- **Revoke compromised keys immediately.** If you suspect a key has been exposed, revoke it from the dashboard right away and generate a replacement before updating your integrations.

<Tip>
  Store your API key in a dedicated secrets manager — such as AWS Secrets Manager, HashiCorp Vault, or your CI/CD platform's built-in secrets store — so it never touches your codebase directly.
</Tip>

## Authentication Errors

If the API cannot verify your identity, it returns `HTTP 401 Unauthorized` with a structured error body explaining the failure.

```json
{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Invalid or missing API key.",
    "status": 401
  }
}
```

<Expandable title="Common causes of 401 errors">
  - **Missing `Bearer ` prefix** — The `Authorization` header value must start with `Bearer ` (including the trailing space) followed by your key. Sending the key alone without this prefix will fail.
  - **Expired or revoked key** — If you rotated or revoked your key in the dashboard, any requests using the old key will be rejected. Update your integration with the new key.
  - **Wrong project key** — If you have multiple projects or workspaces in Pilot Status, make sure you are using the API key that belongs to the correct project.
</Expandable>

If you receive a `401` and your key looks correct, regenerate it from **Dashboard → Settings → API Keys** and redeploy your application with the updated value.
