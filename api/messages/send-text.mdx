# Send a text message

Free-form text is one of the three mutually exclusive modes of the **single** send endpoint — there is no separate `/messages/text` endpoint:

```text
POST https://pilotstatus.com.br/v1/messages/send
```

## Request

Authenticate with your **number-scoped** API key in the `x-api-key` header.

<ParamField body="destinationNumber" type="string" required>
  Destination phone in **E.164** with a leading `+` (e.g., `+5511999999999`).
</ParamField>

<ParamField body="text" type="string" required>
  Body of the free-form message. Mutually exclusive with `templateId` and with direct media (`media` \+ `mediaType`).
</ParamField>

<ParamField body="deliverAt" type="string">
  Optional ISO 8601 datetime to schedule the send.
</ParamField>

```bash
curl -X POST "https://pilotstatus.com.br/v1/messages/send" \
  -H "Content-Type: application/json" \
  -H "x-api-key: ps_your_key_here" \
  -d '{
    "destinationNumber": "+5511999999999",
    "text": "Hi! Your delivery has been confirmed for tomorrow."
  }'
```

## Response (202)

```json
{
  "id": "msg_abc",
  "correlationId": "corr_123",
  "status": "QUEUED"
}
```

The `202` **`id`** is the same value as **`data.id`** on the delivery webhooks (`message.sent`, `message.delivered`, `message.read`, `message.failed`) — see [Webhook events](/api/webhooks/events). `correlationId` is generated by the platform and repeats on correlated events.

## Notes

- Free-form text uses **no template** — nothing needs to be approved beforehand.
- On **Meta Cloud API** numbers, free-form text only works inside the WhatsApp **24h conversation window**; outside it the send fails asynchronously with `META_OUTSIDE_24H_WINDOW` on the `message.failed` webhook — use an approved template instead.
- For the **full parameter list** (destinations, scheduling, labels, buttons, header/footer), see [Send a Message](/api/messages/send).
