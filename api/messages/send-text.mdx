The Send Text endpoint lets you deliver plain text messages to any WhatsApp number directly from your connected Pilot Status number. You provide the source number, the recipient, and the message content — Pilot Status handles delivery and returns a message ID you can use to track the conversation.

`POST https://api.pilotstatus.com.br/v1/messages/text`

## Request body

<ParamField body="number_id" type="string" required>
  The ID of the connected WhatsApp number you want to send from. Retrieve available IDs from [GET /numbers](/api/numbers/list).
</ParamField>

<ParamField body="to" type="string" required>
  Recipient phone number including the country code, without a leading `+`. For example, a Brazilian number would be `5511999999999`.
</ParamField>

<ParamField body="text" type="string" required>
  The message body. WhatsApp formatting is supported: `*bold*`, `_italic_`, `~strikethrough~`, and ` ```monospace``` `.
</ParamField>

## Examples

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api.pilotstatus.com.br/v1/messages/text \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "number_id": "num_abc123",
    "to": "5511999999999",
    "text": "Hello! Your order has been *confirmed* and is on its way."
  }'
```

```javascript Node.js
const response = await fetch(
  "https://api.pilotstatus.com.br/v1/messages/text",
  {
    method: "POST",
    headers: {
      Authorization: "Bearer YOUR_API_KEY",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      number_id: "num_abc123",
      to: "5511999999999",
      text: "Hello! Your order has been *confirmed* and is on its way.",
    }),
  }
);

const data = await response.json();
console.log(data);
```

</CodeGroup>

## Response

A successful request returns `200 OK` with the following fields.

<ResponseField name="id" type="string">
  Unique identifier for the sent message. Store this value if you need to correlate delivery webhooks back to a specific send request.
</ResponseField>

<ResponseField name="status" type="string">
  Initial delivery status. Always `"sent"` on a successful response — subsequent status updates are delivered via webhooks.
</ResponseField>

<ResponseField name="to" type="string">
  The recipient phone number exactly as you provided it in the request.
</ResponseField>

<ResponseField name="timestamp" type="string">
  ISO 8601 timestamp recording when the message was accepted for delivery (e.g., `"2024-01-15T09:45:00Z"`).
</ResponseField>

## Error codes

| Status | Meaning |
|--------|---------|
| `400` | Invalid phone number format — check that `to` contains only digits and includes the country code. |
| `401` | Missing or invalid API key — verify your `Authorization` header. |
| `422` | A required field (`number_id`, `to`, or `text`) is absent from the request body. |
