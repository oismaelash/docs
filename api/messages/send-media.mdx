The Send Media endpoint lets you deliver rich media files — images, videos, audio clips, and documents — to any WhatsApp number from your connected Pilot Status number. You point the API to a publicly accessible URL, and Pilot Status fetches, validates, and forwards the file to WhatsApp on your behalf.

`POST https://api.pilotstatus.com.br/v1/messages/media`

## Request body

<ParamField body="number_id" type="string" required>
  The ID of the connected WhatsApp number to send from. Retrieve available IDs from [GET /numbers](/api/numbers/list).
</ParamField>

<ParamField body="to" type="string" required>
  Recipient phone number including the country code, without a leading `+` (e.g., `5511999999999`).
</ParamField>

<ParamField body="type" type="string" required>
  Media category. Must be one of: `image`, `video`, `audio`, `document`.
</ParamField>

<ParamField body="url" type="string" required>
  A publicly accessible URL pointing to the media file. The URL must be reachable without authentication — Pilot Status downloads the file and forwards it to WhatsApp.
</ParamField>

<ParamField body="caption" type="string">
  Optional caption text displayed below the media in the recipient's chat. Not supported for `audio` type.
</ParamField>

<ParamField body="filename" type="string">
  For `document` type only — the filename shown to the recipient (e.g., `invoice_1024.pdf`). If omitted, the filename is inferred from the URL.
</ParamField>

## Examples

<CodeGroup>

```bash cURL — Image
curl --request POST \
  --url https://api.pilotstatus.com.br/v1/messages/media \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "number_id": "num_abc123",
    "to": "5511999999999",
    "type": "image",
    "url": "https://example.com/assets/promo-banner.jpg",
    "caption": "Check out our latest promotion!"
  }'
```

```bash cURL — Document
curl --request POST \
  --url https://api.pilotstatus.com.br/v1/messages/media \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "number_id": "num_abc123",
    "to": "5511999999999",
    "type": "document",
    "url": "https://example.com/docs/invoice_1024.pdf",
    "caption": "Your invoice for order #1024.",
    "filename": "invoice_1024.pdf"
  }'
```

</CodeGroup>

## Supported formats

| Type | Formats | Max size |
|------|---------|----------|
| `image` | JPG, PNG, WEBP | 5 MB |
| `video` | MP4, 3GP | 16 MB |
| `audio` | MP3, OGG, AAC | 16 MB |
| `document` | PDF, DOCX, XLSX, PPTX | 100 MB |

<Note>
  Media URLs must be publicly accessible without authentication. Pilot Status downloads and forwards them to WhatsApp. If your files are behind a private CDN or require signed URLs, ensure the link remains valid for at least 60 seconds after the request is made.
</Note>

## Response

A successful request returns `200 OK` with the following fields.

<ResponseField name="id" type="string">
  Unique identifier for the sent message.
</ResponseField>

<ResponseField name="status" type="string">
  Initial delivery status. Always `"sent"` on a successful response.
</ResponseField>

<ResponseField name="to" type="string">
  The recipient phone number as provided in the request.
</ResponseField>

<ResponseField name="timestamp" type="string">
  ISO 8601 timestamp recording when the message was accepted for delivery (e.g., `"2024-01-15T09:45:00Z"`).
</ResponseField>
