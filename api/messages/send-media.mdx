# Send a media message

Direct media is one of the three mutually exclusive modes of the **single** send endpoint — there is no separate `/messages/media` endpoint:

```text
POST https://pilotstatus.com.br/v1/messages/send
```

## Request

Authenticate with your **number-scoped** API key in the `x-api-key` header. Send `media` \+ `mediaType` **without** `templateId` and **without** `text`.

<ParamField body="destinationNumber" type="string" required>
  Destination phone in **E.164** with a leading `+` (e.g., `+5511999999999`).
</ParamField>

<ParamField body="media" type="string" required>
  A **public http(s) URL** or a **base64 data URI** (e.g. `data:image/png;base64,...`). Base64 works on **Meta Cloud API** numbers; **unofficial (Pilot Status web)** numbers require a public URL.
</ParamField>

<ParamField body="mediaType" type="string" required>
  `image`, `video`, `document`, or `audio`.
</ParamField>

<ParamField body="caption" type="string">
  Optional caption for `image`, `video`, and `document` — **not** supported for `audio`.
</ParamField>

<CodeGroup>

```bash Image with caption
curl -X POST "https://pilotstatus.com.br/v1/messages/send" \
  -H "Content-Type: application/json" \
  -H "x-api-key: ps_your_key_here" \
  -d '{
    "destinationNumber": "+5511999999999",
    "media": "https://cdn.example.com/photo.jpg",
    "mediaType": "image",
    "caption": "Here is your receipt"
  }'
```

```bash Voice note (audio)
curl -X POST "https://pilotstatus.com.br/v1/messages/send" \
  -H "Content-Type: application/json" \
  -H "x-api-key: ps_your_key_here" \
  -d '{
    "destinationNumber": "+5511999999999",
    "media": "https://cdn.example.com/voice.ogg",
    "mediaType": "audio"
  }'
```

</CodeGroup>

## Response (202)

```json
{
  "id": "msg_abc",
  "correlationId": "corr_123",
  "status": "QUEUED"
}
```

## Notes

- `audio` is always delivered as a WhatsApp **voice note (PTT)**.
- In this mode `buttons`, `header`, `footer`, and `variables` are **not allowed** — to combine media with buttons, use a template send (see [Send Interactive](/api/messages/send-interactive)).
- For formats, size limits, and provider differences, see the [Media messages guide](/guides/media-messages); for the full parameter list, see [Send a Message](/api/messages/send).
