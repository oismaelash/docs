Pilot Status fires a webhook to your configured endpoint for every event that occurs on your numbers — including inbound messages, delivery receipts, read receipts, failures, and number connection changes. Each request is an HTTP POST with a JSON body sent in milliseconds of the event occurring.

## Common payload fields

Every webhook payload includes the following top-level fields, regardless of event type.

<ParamField body="event" type="string" required>
  The event type identifier. Possible values: `message.received`, `message.sent`, `message.delivered`, `message.read`, `message.failed`, `number.connected`, `number.disconnected`.
</ParamField>

<ParamField body="timestamp" type="string" required>
  ISO 8601 timestamp of when the event occurred, e.g. `2024-01-15T09:41:00Z`.
</ParamField>

<ParamField body="number_id" type="string" required>
  The ID of the number that triggered the event. Use this to identify which of your connected numbers the event belongs to.
</ParamField>

## message.received

Pilot Status fires `message.received` when an inbound message arrives on one of your numbers. The `message` object contains the full details of the incoming message.

<Expandable title="message object fields">
  <ResponseField name="message.id" type="string">
    Unique identifier for the message.
  </ResponseField>
  <ResponseField name="message.from" type="string">
    The sender's phone number in E.164 format (without the `+` prefix).
  </ResponseField>
  <ResponseField name="message.type" type="string">
    The message content type. One of: `text`, `image`, `video`, `audio`, `document`, `button_reply`, `location`, `sticker`.
  </ResponseField>
  <ResponseField name="message.text" type="string">
    The message text body. Present when `type` is `text`.
  </ResponseField>
  <ResponseField name="message.timestamp" type="string">
    ISO 8601 timestamp of when the message was sent by the contact.
  </ResponseField>
</Expandable>

```json
{
  "event": "message.received",
  "timestamp": "2024-01-15T09:41:00Z",
  "number_id": "num_abc123",
  "message": {
    "id": "msg_xyz789",
    "from": "5511999999999",
    "type": "text",
    "text": "Hello!",
    "timestamp": "2024-01-15T09:41:00Z"
  }
}
```

## message.sent

Pilot Status fires `message.sent` when an outbound message has been successfully submitted to WhatsApp for delivery. This event confirms that Pilot Status accepted and handed off the message — delivery to the recipient's device is confirmed separately by `message.delivered`.

<Expandable title="Payload fields">
  <ResponseField name="message_id" type="string">
    The ID of the message that was sent.
  </ResponseField>
  <ResponseField name="to" type="string">
    The recipient's phone number in E.164 format (without the `+` prefix).
  </ResponseField>
</Expandable>

```json
{
  "event": "message.sent",
  "timestamp": "2024-01-15T09:41:01Z",
  "number_id": "num_abc123",
  "message_id": "msg_xyz789",
  "to": "5511999999999"
}
```

---

## message.delivered and message.read

Pilot Status fires `message.delivered` when the recipient's device confirms delivery, and `message.read` when the recipient opens the message. Both events share the same payload shape.

<Expandable title="Payload fields">
  <ResponseField name="message_id" type="string">
    The ID of the message that was delivered or read.
  </ResponseField>
  <ResponseField name="to" type="string">
    The recipient's phone number in E.164 format (without the `+` prefix).
  </ResponseField>
</Expandable>

```json
{
  "event": "message.delivered",
  "timestamp": "2024-01-15T09:41:05Z",
  "number_id": "num_abc123",
  "message_id": "msg_xyz789",
  "to": "5511999999999"
}
```

Replace `"event": "message.delivered"` with `"event": "message.read"` for read receipts — all other fields remain identical.

## message.failed

Pilot Status fires `message.failed` when a message could not be delivered. The payload includes an error code and a human-readable description so you can diagnose the issue programmatically.

<Expandable title="Payload fields">
  <ResponseField name="message_id" type="string">
    The ID of the message that failed to deliver.
  </ResponseField>
  <ResponseField name="to" type="string">
    The intended recipient's phone number.
  </ResponseField>
  <ResponseField name="error_code" type="string">
    A machine-readable error code identifying the failure reason, e.g. `invalid_number` or `rate_limit_exceeded`.
  </ResponseField>
  <ResponseField name="error_message" type="string">
    A human-readable description of the error.
  </ResponseField>
</Expandable>

```json
{
  "event": "message.failed",
  "timestamp": "2024-01-15T09:42:00Z",
  "number_id": "num_abc123",
  "message_id": "msg_xyz790",
  "to": "5511888888888",
  "error_code": "invalid_number",
  "error_message": "The destination number is not registered on WhatsApp."
}
```

## number.connected and number.disconnected

Pilot Status fires `number.connected` when a number successfully connects to WhatsApp and `number.disconnected` when it goes offline. Use these events to monitor the health of your connected numbers and trigger alerts or reconnection flows.

<Expandable title="Payload fields">
  <ResponseField name="phone_number" type="string">
    The phone number in E.164 format (without the `+` prefix) that changed connection state.
  </ResponseField>
</Expandable>

```json
{
  "event": "number.connected",
  "timestamp": "2024-01-15T08:00:00Z",
  "number_id": "num_abc123",
  "phone_number": "5511999999999"
}
```

```json
{
  "event": "number.disconnected",
  "timestamp": "2024-01-15T10:30:00Z",
  "number_id": "num_abc123",
  "phone_number": "5511999999999"
}
```

---

<Tip>
  Always respond with HTTP 200 immediately. If your handler takes more than 5 seconds, Pilot Status will retry.
</Tip>
