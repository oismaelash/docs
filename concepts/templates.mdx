Templates let you define reusable message structures once and personalize them at send time using dynamic variables. Instead of constructing message text from scratch in your code, you reference a named template and supply the values that fill in the blanks — Pilot Status handles the rest. For the Official API, templates must pass Meta's review process before you can use them in production. For the Unofficial API, you can create and send templates freely with no approval step.

## Template Syntax

Variables inside a template are written with double curly braces around the variable name. You can use any variable name that makes the template readable and maintainable.

```
Hello {{name}}! Your order #{{order_id}} has been confirmed. Track it here: {{tracking_url}}
```

When you send this template, you supply a `variables` object that maps each placeholder to its runtime value. Pilot Status substitutes the values before delivering the message to the recipient.

## Official API Templates

Because the Official API routes through Meta's infrastructure, every template must be submitted to Meta for review and approved before it can be sent to users. You assign each template a **category** at submission time, and Meta evaluates it accordingly.

  <Card title="OTP" icon="key">

  </Card>
  <Card title="Utility" icon="bell">

  <Card title="Marketing" icon="bullhorn">

  </Card>

To create a new Official API template, navigate to **Dashboard → Templates → New Template**. You will:

1. Enter a template name (lowercase, underscores only — this is the name you reference in API calls).
2. Select a category: **OTP**, **Utility**, or **Marketing**.
3. Write the message body, inserting `{{variable_name}}` placeholders where dynamic content should appear.
4. Optionally add buttons (see [Interactive Buttons](#interactive-buttons) below).
5. Submit the template. Meta's review system typically returns a decision within a few minutes to a few hours.

Once Meta approves the template, its status changes to **Active** in your dashboard and you can begin sending it immediately.

<Note>
  Official API template approval typically takes a few minutes to a few hours via Meta's review system.
</Note>

## Unofficial API Templates

When you use the Unofficial API, there is no pre-approval process. You can create a template in the dashboard or define it inline in your API call and send it immediately — no waiting, no category selection, no Meta review. Variables work exactly the same way using the `{{variable_name}}` syntax.

This makes the Unofficial API the fastest path for prototyping message flows, building internal tooling, or shipping features when you need flexibility before your Official API templates are approved.

## Interactive Buttons

Both Official and Unofficial API templates support interactive buttons that recipients can tap directly inside the WhatsApp chat. You can include up to **3 buttons** per message, choosing from two button types:

<Tabs>
  <Tab title="Quick Reply">
    Quick reply buttons display a tappable text label. When the recipient taps one, WhatsApp sends that label back as a reply message — your webhook receives it as a `message.received` event. Use quick replies to collect structured responses without asking the user to type.

    **Examples**: "Confirm", "Cancel", "Yes, send me updates"
  </Tab>
  <Tab title="URL Button">
    URL buttons open a web page when tapped. The URL can include template variables, letting you generate personalized deep links for each recipient.

    **Example**: "Track order" → `https://example.com/track/{{order_id}}`
  </Tab>
</Tabs>

Here is an example API request that sends a template with variables:

```json
{
  "number_id": "num_abc123",
  "to": "5511999999999",
  "template": {
    "name": "order_confirmation",
    "variables": {
      "name": "Ana",
      "order_id": "1024"
    }
  }
}
```

Pilot Status looks up the `order_confirmation` template, substitutes `Ana` for `{{name}}` and `1024` for `{{order_id}}`, and delivers the final message to the recipient's WhatsApp.
